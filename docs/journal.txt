- Vergleich: Bankdrücken KH 55/80/105/130 (gesamt)

- +30% vs Langhantel (maschinengeführt = stabiler)
- Höher als freie Gewichte
- Vergleich: Bankdrücken LH 55/80/105/130

**Farmers Carry:**
- Griffkraft-limitiert (nicht Beinkraft)
- 25-70kg pro Hand realistisch
- Elite = fast Körpergewicht pro Hand

**Pull-Through (Kabelzug):**
- Deutlich leichter als Kreuzheben
- Fokus auf Hüftstreckung, nicht maximales Gewicht
- ~30-40% von Kreuzheben-Standards

## Fix: Brustpresse Maschine in Equipment-Auswahl fehlte

**Datum:** 2026-02-17
**Branch:** feature/phase-5-3-ai-ml-testing
**Commit:** 4945725

### Problem
"Brustpresse Maschine" erschien NICHT in der Equipment-Auswahl-UI
(/equipment/), obwohl sie:
- In der Datenbank existiert (Equipment-Objekt vorhanden)
- Mit der Übung verknüpft ist (via Admin verifiziert)
- In EQUIPMENT_CHOICES definiert ist (constants.py)

### Ursache
**View-Kategorisierung fehlte!**

In `core/views/exercise_management.py` (Zeile 56-67) wird die Equipment-UI
in Kategorien aufgeteilt. Die "Maschinen"-Kategorie hatte:
```python
"equipment": [
    "KABELZUG",
    "BEINPRESSE",
    "LEG_CURL",
    "LEG_EXT",
    "HACKENSCHMIDT",
    "RUDERMASCHINE",
    "ADDUKTOREN_MASCHINE",
    "ABDUKTOREN_MASCHINE",
    # BRUSTPRESSE_MASCHINE FEHLTE HIER!
],
```

**Equipment die NICHT in einer Kategorie sind, werden NICHT angezeigt!**

### Lösung
`BRUSTPRESSE_MASCHINE` zur Maschinen-Liste hinzugefügt (Zeile 65):
```python
"equipment": [
    "KABELZUG",
    "BEINPRESSE",
    "LEG_CURL",
    "LEG_EXT",
    "HACKENSCHMIDT",
    "RUDERMASCHINE",
    "ADDUKTOREN_MASCHINE",
    "ABDUKTOREN_MASCHINE",
    "BRUSTPRESSE_MASCHINE",  # ← NEU
],
```

### Verifikation
Nach dem Fix:
- Brustpresse Maschine erscheint in der "Maschinen"-Kategorie (gelb, Gear-Icon)
- Kann vom User an-/abgewählt werden
- Funktioniert sowohl lokal (127.0.0.1:8000) als auch auf dem Server

## Abgeschlossen: fix/timer-inserttag-quote-escaping (20.02.2026)

**Problem:** `startRestTimer is not defined` in Chrome-Konsole (DE), Timer funktionierte nur in EN.

**Ursache:** `{% trans "Wort" %}` innerhalb von `onclick="insertTag('... {% trans "X" %} ...')"` –
die inneren Doppel-Quotes des trans-Tags schlossen das HTML-Attribut vorzeitig.
Dadurch wurde der nachfolgende `<script>`-Block nicht korrekt geparst → alle JS-Funktionen undefiniert.

**Fix in training_session.html:**
- Quick-Tag Buttons: `onclick` entfernt, stattdessen `data-tag="... {% trans 'X' %} ..."` (einfache Quotes im trans-Tag)
- Event-Delegation per `document.addEventListener('click', ...)` auf `#quickTagButtons .quick-tag`
- Placeholder-Text: Apostrophe aus msgid entfernt → `z.B. Letzte 2 Wdh. mit Hilfe, Grip zu eng, PR...`
- django.po: altes msgid mit Apostrophen durch neues ersetzt, EN-Übersetzung angepasst
- compilemessages: django.mo aktualisiert
- .gitignore: debug_timer.py, check_trans.py ergänzt

**Commit:** fix/timer-inserttag-quote-escaping

## In Arbeit: feature/fix-1rm-month-window (22.02.2026)

Branch: feature/fix-1rm-month-window
Datei: core/utils/advanced_stats.py, Zeile ~440

Bug: Im 6-Monats-Fenster der 1RM-Entwicklung ist der letzte Slot (i=5)
  monat_start = heute - 30 Tage
  monat_ende  = monat_start + 30 Tage = heute (timezone.now(), ein Datetime)

Der Filter lautet `einheit__datum__lt=monat_ende`. Da `datum` ein DateField ist,
konvertiert Django `monat_ende` zum Datum-Vergleich. Ein Training von heute hat
datum=date.today(), und `today < today` ist False → der heutige PR landet in
keinem Slot und der letzte Slot bleibt None.

Fix: `datum__lt=monat_ende` → `datum__lte=monat_ende`
Kein Risiko für andere Slots: alle anderen Monatsgrenzen liegen in der
Vergangenheit und haben keine Datenpunkte genau auf dem Grenzwert.

## Abgeschlossen: feature/fix-1rm-month-window (22.02.2026)

Branch: feature/fix-1rm-month-window
Datei: core/utils/advanced_stats.py

Fix: `einheit__datum__lt=monat_ende` → `einheit__datum__lte=monat_ende`

Ein Zeichen. Der letzte Slot des 6-Monats-Fensters hatte monat_ende=heute.
Ein Training von heute hat datum=date.today(). Django vergleicht DateField vs
Datetime und truncated auf Datum: `today < today` ist False → heutiger PR
wurde nie in einen Slot eingetragen → letzter Slot immer None.

Mit `__lte` wird today == today eingeschlossen.
